# Qwen AutoGUI 架构逻辑与数据流

本文档记录了系统的核心逻辑架构，特别是“感知（Perception）”、“日志（Journaling）”与“自我进化（Self-Evolution）”的闭环流程。

---

## 1. 核心组件 (Core Components)

| 组件 | 作用 | 存储位置 |
| :--- | :--- | :--- |
| **Session** | 实时对话历史，包含原始消息和 rolling summary。 | `data/sessions/*.json` |
| **User Profile** | **核心记忆层**：包含 Objective (身份事实) 与 Subjective (主观规则) 记忆。 | `data/user_profile.json` |
| **Scene Memory** | **片段记忆层**：提取的事实性长期记忆（海量归档，支持向量检索）。 | `data/scene_memory.jsonl` |
| **Journal** | 每日流水账日志，是进化的“原料库”。 | `data/journals/YYYY-MM-DD.md` |
| **Diary** | AI 的第一人称感悟，是进化后的“成品”。 | `data/diary/YYYY-MM-DD.md` |
| **Persona** | AI 的性格定义，会随交互动态微调。 | `PERSONA.md` |

---

## 2. 数据流循环 (The Data Loop)

### A. 实时感知阶段 (Perception)
*   **视觉感知**：后台 `ContextManager` 每 5 分钟截屏一次，由多模态模型分析用户状态。
*   **即时写入**：分析结果（状态 + 简评）立即追加到 `data/journals/` 的当日日志中。

### B. 会话存储阶段 (Interaction)
*   **对话持久化**：所有的聊天记录实时保存在 `data/sessions/`。此时对话**不**会立即进入 Journal（为了保持 Journal 的简洁性）。

### C. 跨天/启动进化阶段 (Self-Evolution)
这是系统最核心的逻辑，触发点为：**程序启动时**（发现昨日未总结）或**零点跨天第一次聊天时**。

#### 流程 1：对话汇总 (Chat Summarization)
系统扫描所有 Session 文件，提取指定日期的所有消息，调用 LLM 进行压缩总结，将其作为 `**[聊天总结]**` 写入当日的 Journal 文件。此时，Journal 变得完整，包含了：
1.  **视觉日志**（我在屏幕上看到了什么）
2.  **聊天总结**（我和用户聊了什么）

#### 流程 2：日志进化 (Evolve from Journal)
`SelfEvolution` 模块读取完整的 Journal，执行以下子任务：
1.  **合成日记 (Diary)**：以 AI 第一人称视角，结合“视觉”与“对话”，写下一天的感悟。
2.  **分层记忆路由 (Memory Stratification)**：
    *   **User Profile (Core)**：将新发现的用户固定属性（Objective）或核心规则偏好（Subjective）固化到档案。
    *   **Scene Memory (Episodic)**：将普通的过去事件事实存入片段记忆库供 Top-K 检索。
3.  **知识图谱 (KG)**：提取实体关系三元组。

#### 3. 即时偏好更新 (Real-time Update)
无需等到跨天，当用户在对话中明确下达“以后注意...”等指令时，模型会调用 `update_user_profile` 工具立即更新主观记忆层，确保下一次 Prompt 组装时规则立即生效。

#### 4. 人设微调 (Persona Evolution)
根据最新生成的记忆，AI 评估并更新 `data/interaction_habits.md`（交互习惯沉淀），维持 `PERSONA.md` 源文件的简洁与稳定。

## 3. 关键逻辑路径

### 跨天检测触发器 (`agent.py`)
```python
# _startup_evolution (启动时)
# 寻找[有日志但没日记]的日期 -> 后台线程 -> 触发汇总与进化

# _check_all_triggers (运行时)
# if current_date != last_interaction_date:
#    放入后台线程 -> 执⾏昨天日期的汇总与进化，不阻塞当前用户的下一次问答
```

### 记忆分层与防御机制
为了确保模型在执行长任务时不遗忘用户偏好，记忆被分为三层：
1.  **Objective Memory (客观档案)**：[常驻注入] 姓名、年龄、硬件环境等。
2.  **Subjective Memory (主观偏好)**：[常驻注入] 你的“戒律”与习惯。
3.  **Scene Memory (场景事实)**：[Top-K 检索] 过去的具体事件。

**提取逻辑 (`self_evolution.py`)**：
*   **输入**：人设文件 + 视觉日志 + 聊天总结。
*   **记忆输出**：根据 `EXTRACTION_PROMPT` 的 layer 指令，自动路由至 `user_profile.json` 或 `scene_memory.jsonl`。
*   **知识图谱输出**：抽取的 `<实体-关联-实体>` Triples。

## 4. 手动干预与运维工具

系统在根目录下提供了 `force_evolve.py` 脚本，可用于手动强制修正或重新触发某一天的全部汇总流程。

```bash
# 删除今天已经生成的旧日记
del data\diary\2026-02-19.md

# 清空片段记忆与图谱
del data\scene_memory.jsonl
del data\scene_memory_vectors.jsonl
del data\knowledge_graph.jsonl

# 重新触发进化
python force_evolve.py 2026-02-19
```

---

## 4. 维护说明
- 若要修改视觉密度，调整 `main.py` 中的 `interval_seconds`。
- 若要调整记忆提取逻辑，修改 `self_evolution.py` 中的 `EXTRACTION_PROMPT`。
- 若要彻底重置某个日期的进化结果，删除对应的 `data/diary/YYYY-MM-DD.md` 并重启。
