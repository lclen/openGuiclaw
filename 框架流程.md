# Qwen AutoGUI 架构逻辑与数据流

本文档记录了系统的核心逻辑架构，特别是"感知（Perception）"、"日志（Journaling）"与"自我进化（Self-Evolution）"的闭环流程。

---

## 1. 核心组件 (Core Components)

| 组件 | 作用 | 存储位置 |
| :--- | :--- | :--- |
| **Session** | 实时对话历史，包含聊天消息、`visual_log`（视觉旁白）和 rolling summary。 | `data/sessions/*.json` |
| **User Profile** | **核心记忆层**：包含 Objective (身份事实) 与 Subjective (主观规则) 记忆。 | `data/user_profile.json` |
| **Scene Memory** | **片段记忆层**：提取的事实性长期记忆（海量归档，支持向量检索）。 | `data/scene_memory.jsonl` |
| **Journal** | 每日总结日志，是进化的"原料库"（由每日汇总生成，不再实时写入）。 | `data/journals/YYYY-MM-DD.md` |
| **Diary** | AI 的第一人称感悟，是进化后的"成品"。 | `data/diary/YYYY-MM-DD.md` |
| **Persona** | AI 的性格定义，会随交互动态微调。 | `PERSONA.md` |

## 2. 数据流循环 (The Data Loop)

### A. 实时感知与存储阶段 (Perception & Storage)

视觉感知与聊天记录统一存入 **Session**，形成完整的、带时间戳的交互时间线。

**视觉感知 (`ContextManager`)**:
1. 后台线程每 **5 分钟**截屏一次，由多模态模型（`qwen-vl-plus`）分析用户状态。
2. 分析时注入**历史上下文**：最近 5 条聊天记录 + 最近 4 条视觉感知旁白（均来自当前 Session）。
3. 分析结果（状态 + 简评）以 `role = "visual_log"` 写入**当前的 `session.json`**，与聊天消息存在一起。
4. 若处于"活泼模式"且用户非工作状态，可能触发主动通知推送。

**聊天模型 (`Agent`)**:
1. 每次回复前，从当前 Session 中提取最近 30 条**聊天**消息（过滤掉 `visual_log`，不发送给 OpenAI API）。
2. **Omni-Context**：同时提取当前 Session 中最近 3 条 `visual_log`，注入到 System Prompt，让大脑"感知"用户当前状态。

```
Session JSON 时间线示例:
[15:00] user: "帮我看看这段代码"
[15:01] assistant: "这里有个缩进问题..."
[15:05] visual_log: "💻 working — 用户在 VSCode 编写 Python 代码"
[15:10] user: "好了我去看会B站"
[15:11] visual_log: "🎮 entertainment — 用户在 Bilibili 观看视频"
```

### B. 跨天/启动进化阶段 (Self-Evolution)

触发点：**程序启动时**（发现昨日未总结）或**零点跨天第一次聊天时**。

#### 步骤 1：对话汇总 (`_summarize_day_conversations`)
1. 扫描所有 `data/sessions/*.json`，提取指定日期的全部消息（含 `visual_log`）。
2. **按时间戳排序**，合并多个 Session 为完整的单日时间线。
3. 格式化时，`visual_log` 转换为 `【系统旁白:视觉感知】` 角色，与对话穿插排列。
4. 调用 LLM 将混合时间线压缩为主题总结，写入 `data/journals/YYYY-MM-DD.md`。

此时 Journal 包含了用言语和行为交织的完整一天记录。

#### 步骤 2：日志进化 (`evolve_from_journal`)
`SelfEvolution` 模块读取完整的 Journal，执行：
1. **合成日记 (Diary)**：以 AI 第一人称视角写下一天的感悟。
2. **分层记忆路由**：
   - **User Profile (Core)**：固化用户固定属性与核心规则偏好。
   - **Scene Memory (Episodic)**：将过去事件事实存入片段记忆库。
3. **知识图谱 (KG)**：提取实体关系三元组。

#### 步骤 3：人设微调 (`evolve_persona`)
根据最新记忆，评估并更新 `data/interaction_habits.md`。

## 3. 关键逻辑路径

### 跨天检测触发器 (`agent.py`)
```python
# _startup_evolution (启动时)
# 寻找[有 session 消息但没日记]的日期 -> 后台线程 -> 触发汇总与进化

# _check_all_triggers (运行时)
# if current_date != last_interaction_date:
#    后台线程 -> 对昨天日期执行汇总与进化，不阻塞当前问答
```

### 记忆分层与防御机制
1. **Objective Memory (客观档案)**：[常驻注入] 姓名、年龄、硬件环境等。
2. **Subjective Memory (主观偏好)**：[常驻注入] 用户的"戒律"与习惯。
3. **Scene Memory (场景事实)**：[Top-K 检索] 过去的具体事件。

**即时偏好更新**：当用户说"以后注意..."时，模型立即调用 `update_user_profile` 工具更新档案，无需等到跨天进化。

## 4. 手动干预与运维工具

根目录提供 `force_evolve.py` 脚本，可手动强制触发某一天的全部汇总流程。

```bash
# 删除已有日记（允许重新生成）
del data\diary\2026-02-19.md

# 清空片段记忆与图谱（可选，彻底重置）
del data\scene_memory.jsonl
del data\scene_memory_vectors.jsonl
del data\knowledge_graph.jsonl

# 重新触发进化
python force_evolve.py 2026-02-19
```

## 5. 模型生态与功能分布 (Model Ecology)

系统中同时运行着多个不同定位的 AI 模型，它们各司其职构建了完整的架构：

| 模型分类 | 配置文件项 | 核心作用与代码调用位置 |
| :--- | :--- | :--- |
| **主回复模型**<br>(Chat Model) | `api.model`<br>*(大预言模型)* | **主管前台业务交互**：<br>- `agent.py -> chat()`: 处理日常对话请求流，解析并调用 Tools/Skills。<br>- `agent.py -> _summarize_day_conversations()`: 在跨天或启动时负责对历史多条对话+视觉日志进行粗粒度压缩汇总，生成原始 Journal。 |
| **视觉模型**<br>(Vision Model) | `vision_api.model`<br>*(多模态大模型)* | **主管后台环境感知**：<br>- `context.py -> analyzer_loop()`: 独立后台线程每 5 分钟获取屏幕截图及上下文交由该模型视觉分析用户当前行为状态，产出 `visual_log`。<br>- `agent.py -> add_visual_log / update_visual_log`: 写入 Session 时间线。 |
| **进化模型**<br>(Evolution Model) | `api.evolution_model`<br>*(默认且通常同主模型)* | **主管午夜人格潜意识挂机整理**（集中在 `core/self_evolution.py`）：<br>- `evolve_from_journal()`: 写第一人称日记 (Diary)，提取提取核心档案 (User Profile)，提取片段层记忆 (Scene Memory)。<br>- `_extract_kg_triples()`: 提取知识图谱三元组。<br>- `evolve_persona()`: 反思并修改未来交互习惯与人设 (`interaction_habits.md`)。 |
| **向量模型**<br>(Embedding Model)| `embedding_api.model`<br>*(稠密文本向量模型)* | **主管记忆的高维语义检索计算**：<br>- `vector_memory.py`: 分别在存储、搜索与比对 Scene Memory 相似度时计算高维嵌入。<br>- `agent.py -> _backfill_vectors()`: 后台线程静默补全缺失的记忆向量。<br>- `journal_index.py`: 对历史日记做向量化索引与语义搜索支持。 |

## 6. 技能与插件体系 (Skill & Plugin System)

系统通过三层结构扩展能力，支持从硬编码工具到动态文档自学的全过程：

### A. 内置技能 (Built-in Skills)
- **定义位置**：`core/agent.py -> _register_builtins()`。
- **核心工具**：
    - `remember` / `search_memory`: 长期记忆入库与三合一检索。
    - `create_plan` / `update_plan_step`: 复杂任务的状态机流转。
    - `update_user_profile`: 实时修改用户档案。

### B. 插件体系 (Plugin System)
- **管理器**：`core/plugin_manager.py (PluginManager)`。
- **加载逻辑**：启动时扫描 `plugins/*.py`，调用其 `register(skill_manager)`。
- **热更新**：支持运行时 `/plugins reload` 重新加载 Python 逻辑。
- **生命周期**：插件现在可以追踪并清理自己注册的 Skill，防止卸载后残留。

### C. 文档驱动技能 (SKILL.md / Documentation Skills)
这是最灵活的扩展方式，允许 AI 通过阅读 Markdown 文档来学习如何使用外部 CLI 工具。

1. **扫描与索引**：
    - `agent.py -> _scan_local_skills()`: 扫描 `skills/` 和 `.agents/skills/` 目录。
    - 提取 YAML Metadata（名称、描述、`allowed-tools` 白名单）及 `scripts/` 子目录脚本。
2. **意图匹配与引导**：
    - `agent.py -> _find_relevant_skills(user_query)`: 每次对话时分词匹配描述，命中后主动在提示词中建议用户尝试该技能。
3. **按需学习 (Progressive Disclosure)**：
    - `list_skills`: 向 LLM 展示简要目录及描述。
    - `get_skill_info(name)`: LLM 调用此工具读取完整的 `SKILL.md`，实现"现学现卖"。
4. **动态安装**：
    - `install_skill(url)`: 从 Git 或 URL 下载、解析、存储并注册新技能，自动生成 `plugins/remote_*.py` 包装器。

## 7. 维护说明
- **视觉感知频率**：调整 `main.py` 中的 `interval_seconds`（默认 300 秒）。
- **主动交互模式**：运行时输入 `/mode` 切换静默/正常/活泼模式。
- **记忆提取逻辑**：修改 `self_evolution.py` 中的 `EXTRACTION_PROMPT`。
- **彻底重置某日**：删除对应的 `data/diary/YYYY-MM-DD.md` 并重新运行 `force_evolve.py`。

## 8. 3D & VRM 进阶系统逻辑

### A. 视口持久化与热切换 (Persistence & Seamless Switching)
1. **Snapshot 机制**：在 `switchVrmModel` 前，系统会捕捉当前相机的 `position`、`quaternion` 以及模型自身的 `position`/`scale`存入快照。
2. **RAF 延迟应用**：由于 `loadModel` 内部会重置相机，系统使用 `requestAnimationFrame` 延迟 2 帧，在渲染稳定后再将快照参数强行覆盖，实现视觉上的"无缝"继承。
3. **动画遗留继承**：切换模型后，系统会自动读取 `vrm-animation.js` 中记录的 `_lastVrmaUrl` 并重播，保持动作连贯性。

### B. VRM 商店与资源下载流 (Store & Async Download)
1. **资源感知**：
   - 模型目录：`static/models/*.vrm`
   - 动画目录：`static/vrm/animation/*.vrma`
2. **下载链路**：
   - 前端发起 `POST /api/store/download`。
   - 后端 `server.py` 使用 `httpx` 异步流式写入文件至对应目录。
   - 前端通过列表差异比对（如 `isAnimationDownloaded`）实时更新商店 UI 为"已下载"状态，并动态刷新全局快捷动作按钮。
