## 6. 数据存储现代化 (Storage Modernization)
> 当前 sessions、scene_memory、plans、向量索引等均以 JSON/JSONL 平文件存储。随着历史数据积累，检索、事务安全和并发访问方面的局限性会日益凸显。

- [ ] **SQLite 统一存储层**: 将 `sessions`、`scene_memory`、`plans`、`user_profile`、`knowledge_graph` 等所有结构化数据迁移至单一 SQLite 数据库（`data/agent.db`），保留 JSONL 作为可读备份/迁移工具。
  - 好处：支持 SQL 查询、事务、并发安全；`/sessions` 支持按关键词/日期过滤；history 可分页加载，不受文件行数影响。
- [ ] **向量索引迁移至 SQLite-VSS / ChromaDB**: 替换当前的 JSONL 向量存储（`scene_memory_vectors.jsonl`），使用支持 ANN（近似最近邻）的向量数据库，提升大规模语义检索的速度与精度。
  - 候选方案：[sqlite-vec](https://github.com/asg017/sqlite-vec)（无额外进程）、[ChromaDB](https://www.trychroma.com/)（本地模式）、[Qdrant](https://qdrant.tech/)（本地模式）。
- [ ] **数据迁移与兼容脚本**: 提供 `tools/migrate_to_db.py` 脚本，将现有 JSON/JSONL 文件一键导入数据库，支持从旧版本一键升级，无需重新生成记忆或向量。

## 7. 竞品对比与核心特性借鉴 (Competitor Analysis & Inspirations)
> 基于对兄弟开源项目（memU, OpenAkita, nanobot）的调研，总结出的可供 `qwen_autogui` 借鉴的优化方向：

- [x] **7.1 自我诊断与容错系统 (Self-Healing & Error Recovery) - 借鉴 `OpenAkita`**
  - **现状**：出错时依赖屏幕解析大模型的判断（可能会像之前一样因为 Payloads 过大导致连锁崩溃）。
  - **优化**：引入后台“死循环跳出（Never-Give-Up）”和“自我体检（Self-check）”机制。当检测到连续失败或底层运行报错（如 API 限制、元素无法找到）时，触发 LogAnalyzer 进行底层日志分析，自动回退策略或生成修复代码。
- [ ] **7.2 层次化前瞻性记忆 (Proactive Context Loading) - 借鉴 `memU`**
  - **现状**：记忆目前侧重于日志分块和图谱三元组记录，检索时较为被动。
  - **优化**：将记忆结构化为（资源 -> 事实实体 -> 行为模式分类）。Agent 能够通过后台监控静默提取“你的习惯”，例如“用户通常在写 Python 时喜欢开启副屏终端”，从而在前瞻主动互动时**不仅仅是寒暄**，还能“提前帮你推算下一步可能要打开的文档或网址（Intent Prediction）”。
- [ ] **7.3 极简多模型路由与接入层 (LLM Provider Registry) - 借鉴 `nanobot`**
  - **现状**：目前 `config.json` 中独立配置 api / vision / embedding 等模型。
  - **优化**：梳理一个标准化的 `Provider Registry`，统一调度不同的底座（DeepSeek / Claude / 本地 vLLM）。对于耗费大量 token 的视觉分析，可一键无缝回退到廉价的轻量模型，保证 GUI 操作的经济性和稳定性。
- [ ] **7.4 即时通讯集成 (IM Channels / Webhooks) - 借鉴 `nanobot` / `OpenAkita`**
  - **现状**：计划中已有 FastAPI 网关和原生 Web UI。但对于电脑控制，远程能力极具杀伤力。
  - **优化**：通过简单的 Token 接入，将控制端绑定到 Telegram、微信企业版或飞书。让你可以走在路上给 Agent 发一条微信：“帮我把电脑上昨晚写的那个文档整理成 PPT 发给我”。
- [ ] **7.5 严格的动作执行校验 (Plan Verification)**
  - **现状**：使用 pyautogui 直接下达执行命令，缺乏像 Web 网页操作那样的精确 DOM 树反馈。
  - **优化**：建立“Plan -> Act -> Verify(截屏校验)”的强闭环。在上一个点击动作真正生效（画面跳转或提示消失）前，绝不盲目执行下一步，以此大幅提升电脑操作的稳健性。
