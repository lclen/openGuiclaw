# UI 与钉钉通道优化技术文档 (2026-03-08)

本文档记录了针对 `openGuiclaw` 系统的界面交互体验及钉钉通道图片渲染能力的重大优化。

## 1. 历史面板交互优化 (UI/UX)

### 1.1 面板常开逻辑
*   **优化前**：在历史面板中选择某个会话后，面板会自动关闭，导致用户想连续查看多个会话时需要频繁重复打开。
*   **优化后**：修改了 `loadSession` 的调用逻辑，点击会话列表项时面板将保持开启状态，直至用户手动关闭或点击外部区域。

### 1.2 瞬间定位优化
*   **视觉改进**：移除了聊天容器的 `scroll-smooth` 类。
*   **效果**：切换历史会话时，聊天记录不再从顶部缓慢滚动到底部，而是瞬间定位到最新消息，大幅提升了操作响应感。

### 1.3 外部点击自动关闭
*   **交互逻辑**：为 `chat_area` 的主列添加了点击监听（使用 `@click.stop` 避免按钮点击冲突）。
*   **效果**：当历史面板展开时，点击主聊天区域会自动收起面板，符合现代 Web 应用的沉浸式操作直觉。

## 2. 钉钉通道图片显示优化

### 2.1 自动媒体上传机制
*   **痛点**：钉钉作为公网平台，无法直接读取用户本地 `data/screenshots/` 目录下的图片。
*   **解决建议**：在 `DingTalkAdapter` 中实现了 `_resolve_local_images` 预处理器。
*   **技术细节**：
    *   拦截流出的 Markdown 消息。
    *   正则匹配其中的本地路径（支持 Windows 反斜杠格式）。
    *   自动将本地文件上传至钉钉媒体服务器以换取 `media_id`。
    *   将 Markdown 中的原始路径替换为钉钉内部可渲染的 `![alt](media_id)` 语法。

### 2.2 服务器路由映射
*   **前端同步**：优化了网页前端的 `mdRender` 函数，自动将 `/screenshots/` 相对路径映射到服务器静态资源路由，确保本地网页端和远端钉钉端看到的是完全一致的图片。

## 3. 技能输出标准化 (get_screenshot)
*   **优化**：将 `get_screenshot` 技能的返回格式从“纯文本链接”优化为“Markdown 图片嵌入 + 相对路径”。
*   **意义**：不再写死 `localhost:8000` 端口（避免端口变动导致失效），且直接触发前端和钉钉的图片渲染逻辑。
